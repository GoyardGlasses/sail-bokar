OVERVIEW
================================================================================

SAIL Bokaro Optimizer is a comprehensive logistics optimization system for 
SAIL Bokaro Steel Plant. It combines advanced optimization algorithms with 
machine learning to provide optimal dispatch plans for rail and road transport.

Key Features:
- Real-time optimization engine for dispatch planning
- Multi-modal transport optimization (Rail + Road)
- ML-based delay and cost prediction
- Demand forecasting
- Beautiful, responsive desktop UI
- Cross-platform support (Windows, macOS, Linux)
- Offline-capable with bundled backend

================================================================================
SYSTEM COMPONENTS
================================================================================

1. BACKEND (FastAPI)
   - Optimization engine (Linear Programming)
   - ML model inference pipeline
   - RESTful API endpoints
   - Health monitoring
   - Metrics collection

2. FRONTEND (React + Vite)
   - Dashboard with KPI cards
   - Optimization interface
   - Results visualization
   - Admin panel
   - Real-time health monitoring

3. ELECTRON WRAPPER
   - Desktop application packaging
   - Backend process management
   - Secure IPC communication
   - Auto-update support
   - Cross-platform installers

4. ML MODELS
   - Demand forecasting (XGBoost)
   - Delay prediction (Random Forest)
   - Throughput prediction (Gradient Boosting)
   - Cost estimation (Linear Regression)
   - Mode classification (Logistic Regression)

INSTALLATION
================================================================================

WINDOWS
1. Download: SAIL-Bokaro-Optimizer-Setup-1.0.0.exe
2. Run installer
3. Follow on-screen prompts
4. Launch from Start Menu

MACOS
1. Download: SAIL-Bokaro-Optimizer-1.0.0.dmg
2. Open DMG file
3. Drag app to Applications folder
4. Launch from Applications

LINUX (Ubuntu/Debian)
1. Download: sail-bokaro-optimizer-1.0.0.deb
2. Run: sudo dpkg -i sail-bokaro-optimizer-1.0.0.deb
3. Launch: sail-bokaro-optimizer

LINUX (AppImage)
1. Download: SAIL-Bokaro-Optimizer-1.0.0.AppImage
2. Run: chmod +x SAIL-Bokaro-Optimizer-1.0.0.AppImage
3. Execute: ./SAIL-Bokaro-Optimizer-1.0.0.AppImage

================================================================================
GETTING STARTED
================================================================================

1. LAUNCH THE APP
   - App will start backend automatically
   - Wait for "Backend Connected" indicator

2. VIEW DASHBOARD
   - See real-time KPI metrics
   - Monitor system health

3. RUN OPTIMIZATION
   - Click "Optimize" in sidebar
   - Set available rakes and trucks
   - Click "Run Optimization"
   - Wait for results (30-60 seconds)

4. VIEW RESULTS
   - See rake and truck allocation
   - View cost breakdown
   - Export plan as JSON

5. MANAGE SYSTEM
   - Click "Admin" for system controls
   - Reload models if needed
   - View system metrics

================================================================================
API ENDPOINTS
================================================================================

HEALTH & META
GET  /meta/health              - Backend health check
GET  /meta/metrics             - System metrics
GET  /meta/config              - System configuration
GET  /meta/models              - Loaded ML models
POST /meta/reload-models       - Reload all models

PREDICTIONS
POST /predict/demand           - Demand forecasting
POST /predict/rake-availability - Rake availability prediction
POST /predict/delay            - Delay prediction
POST /predict/throughput       - Throughput prediction
POST /predict/cost             - Cost estimation
POST /predict/mode             - Mode classification

OPTIMIZATION
POST /optimize/dispatch        - Run optimization

================================================================================
KNOWN ISSUES & LIMITATIONS
================================================================================

1. PERFORMANCE
   - Large problems (>1000 orders) may take 2-5 minutes
   - Optimization quality improves with more data

2. MODELS
   - ML models are trained on historical SAIL Bokaro data
   - Accuracy may vary with different input patterns
   - Models should be retrained quarterly with new data

3. BACKEND
   - Backend requires Python 3.10+ runtime
   - Port 8000 must be available
   - Some antivirus software may flag the binary

4. UI
   - Some animations may be slow on older hardware
   - Responsive design optimized for 1024x768 minimum

5. DATA
   - Sample data is synthetic for demo purposes
   - Real data should be imported via CSV upload

================================================================================
TROUBLESHOOTING
================================================================================

APP WON'T LAUNCH
- Check system requirements
- Disable antivirus temporarily
- Try portable version (Windows)
- Check disk space (500MB minimum)

BACKEND WON'T START
- Check port 8000 is available
- Check Python runtime is installed
- View logs: Admin → View Logs
- Try "Restart Backend" in Admin panel

OPTIMIZATION FAILS
- Check backend health indicator
- Try with smaller parameters
- Check sample data is valid
- View error logs

SLOW PERFORMANCE
- Close other applications
- Check system resources
- Increase RAM allocation
- Restart the application
